PHẦN II – PHÂN TÍCH TÌNH HUỐNG 2,0 điểm

Bối cảnh: Công ty XYZ vận hành một website thương mại điện tử (dùng PHP/MySQL) cho phép khách hàng tìm kiếm sản phẩm qua tham số ?q= và đăng nhập bằng form. Gần đây, đội ngũ phát hiện log ghi nhận các truy vấn SQL bất thường và cảnh báo từ WAF về tấn công XSS. Ngoài ra, server thường xuyên quá tải vào giờ cao điểm do một lượng lớn request từ nhiều IP khác nhau gửi đến trang tìm kiếm.

Yêu cầu: Phân tích rủi ro theo mô hình CIA (Confidentiality, Integrity, Availability). Chỉ ra ít nhất 3 nguy cơ tương ứng với từng khía cạnh và đề xuất biện pháp giảm thiểu.

PHẦN III – PHÂN TÍCH MÃ NGUỒN 2,0 điểm

Đoạn code xử lý đăng nhập trong hệ thống cũ:

// Kết nối MySQL (đã có $conn) $username = $_POST['username']; $password = $_POST['password']; $query = "SELECT * FROM users WHERE username = '$username' AND password = '$password'"; $result = mysqli_query($conn, $query); if (mysqli_num_rows($result) > 0) { $_SESSION['user'] = $username; echo "Đăng nhập thành công!"; } else { echo "Sai tài khoản hoặc mật khẩu"; }
  1. Chỉ ra lỗ hổng bảo mật trong đoạn code trên (0,5 điểm).
  2. Viết lại đoạn code theo chuẩn an toàn sử dụng Prepared Statement (1,0 điểm).
  3. Giải thích nguyên lý phòng chống mà kỹ thuật đó mang lại (0,5 điểm).
